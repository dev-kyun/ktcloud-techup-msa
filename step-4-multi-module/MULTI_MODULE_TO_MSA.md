# 멀티 모듈에서 MSA로 확장하기

이 문서는 현재 멀티 모듈 구조를 MSA로 확장하는 관점에서 어떤 점이 이어지는지 설명.  
지금은 단일 레포/단일 빌드에서 여러 실행 모듈을 관리하고, 이후 서비스로 분리하는 흐름을 목표.

## 현재 구조에서 보이는 MSA 전환 포인트

- **실행 모듈 분리**: `app`, `admin`은 각각 독립 실행 모듈이며, MSA 전환 시 서비스 단위가 될 수 있음
- **공통 모듈 재사용**: `core`, `util`은 여러 모듈에서 재사용되며, 향후 공통 라이브러리나 플랫폼 모듈로 분리 가능
- **모듈 간 의존성**: `app`/`admin`이 `core`/`util`에 의존하는 구조는 서비스 간 공통 의존성을 정리하는 출발점
- **모듈 단위 빌드/배포**: 각 모듈은 개별 빌드가 가능하고, 패키지 단위로 배포 흐름을 분리할 수 있음

## 확장 단계 예시

1. **단일 레포 멀티 모듈**
   - 공통 코드를 모듈로 분리
   - 실행 모듈을 역할별로 분리

2. **서비스별 모듈 강화**
   - `app`/`admin`에 서비스별 도메인 로직 집중
   - 공통 로직은 `core`/`util`에 유지

3. **서비스 분리(멀티 레포 또는 멀티 빌드)**
   - 실행 모듈을 독립 레포/독립 배포 단위로 분리
   - 공통 모듈은 내부 라이브러리로 배포

## 현재 예제에서 할 수 있는 연습

- `app`와 `admin`에 서로 다른 기능 추가
- `core`에 공통 도메인 로직을 추가해 두 모듈에서 재사용
- 모듈별 빌드/실행 흐름을 구분해서 운영
- 모듈 간 참조 불가 ( 코드 격리를 통한 제약 확인 )
