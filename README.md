## 준비
도커 설치

## 사전 지식 

| 항목 | 수준 | 설명 |
|------|------|------|
| Java 기초 | 필수 | 클래스, 패키지, main 메서드 이해 |
| 빌드 과정 | 권장 | 컴파일(.java → .class), 패키징(JAR) 개념 |
| 터미널/CLI | 권장 | 기본 명령어 사용 가능 |
| 배포 경험 | 선택 | 있으면 좋지만 없어도 학습 가능 |
| MSA 개요 | 선택 | 개념만 알면 충분 |

### 빌드 과정 간단 설명
```aiexclude
.java → [컴파일] → .class → [패키징] → .jar(Java ARchive. 압축파일)
```


### MSA 간단 설명
```aiexclude
모놀리식 (1인 식당)                    MSA ( 여러 담당 직원 ) 

┌─────────────────────┐          ┌───────┐ ┌───────┐ ┌───────┐
│                     │          │  주문  │ │  조리  │ │  결제  │
│    1인 식당 사장       │          │  담당  │ │  담당  │ │  담당  │
│                     │          └───────┘ └───────┘ └───────┘
│  주문 + 조리 + 결제    │
│  + 서빙 + 청소        │          ┌───────┐ ┌───────┐
│                     │          │  서빙  │ │  청소  │
│   모든 걸 혼자 처리    │           │  담당  │ │  담당  │
└─────────────────────┘          └───────┘ └───────┘
```
MSA 특징
- 손님이 많아지면 필요한 역할만 늘릴 수 있음.
- 비용이 많이 든다. (인건비, 관리, 소통, ...) 
- 특정 담당이 없어도 돌아갈 수 있다.



## 학습 흐름
```
수동 빌드 이해 → Gradle 자동화 → 멀티모듈 → Docker → MSA

step-1,2        step-3          step-4       step-5,6
    │               │              │            │
    ▼               ▼              ▼            ▼
"빌드가 뭔지"       "자동화 도구"    "모듈 분리"    "컨테이너(도커) 이해"
  직접 체험          사용법         구조 설계     Spring Boot환경 멀티모듈 적용 

```


# 학습 포인트
- MSA 구축을 위한 기반 지식을 단계 별로 학습

# 프로젝트 구조

```
msa/
├── step-1-pure-manual-build/      # 순수 자바 수동 빌드 예제
├── step-2-manual-build/           # 수동 jar + manifest.mf 빌드 예제
├── step-3-gradle-build/           # Gradle 빌드 예제
├── step-4-multi-module/           # Gradle 멀티모듈
├── step-5-spring-boot-single/     # Spring Boot 단일 모듈 + Docker 예제
├── step-6-spring-multi-module/    # Spring Boot 멀티모듈 + Docker 예제
```

# 단계별 학습 내용

| 단계 | 주제          | 핵심 학습 내용                    |
|------|-------------|-----------------------------|
| step-1 | 수동 빌드 기초    | javac, java 명령어로 직접 컴파일/실행  |
| step-2 | JAR 패키징     | jar 명령어, MANIFEST.MF 역할 이해  |
| step-3 | Gradle 기초   | 빌드 자동화, 의존성 관리              |
| step-4 | 멀티모듈        | 모듈 분리, 공통 모듈 재사용            |
| step-5 | Spring Boot | Spring Boot 환경 빌, Docker 기초 |
| step-6 | 드MSA 기초     | 서비스 분리, 독립 관리, Docker 기초    |
